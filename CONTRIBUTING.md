# Contributing to Designer Repository

This guide explains how to properly add or update design assets.

## How to Contribute

1. Make your changes in the appropriate folder:
   - `assets/` for images/icons/illustrations
   - `components/` for UI components
   - `mockups/` for project mockups
   - `prototypes/` for interactive prototypes
   - `docs/` for documentation or guides
   - `branding/` for logos, colors, fonts
   - `templates/` for UI templates
2. Follow naming conventions and versioning guidelines.
3. Commit your changes: `git commit -m "Add your description"`
4. Push the commit on this repository.

## Review Guidelines

- PRs must be reviewed by at least one designer.
- Ensure all files meet style and naming conventions.
- Documentation updates must be clear and concise.

## File Naming & Versioning

- Lowercase, hyphen-separated names: `login-screen-v1.fig`
- Version control: append `v1`, `v2`, `final`, etc.
